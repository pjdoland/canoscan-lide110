# CanoScan LiDE 110 for macOS

A native macOS app for the Canon CanoScan LiDE 110 flatbed scanner. Canon never provided working drivers for modern macOS, so this app wraps the open-source [SANE](http://www.sane-project.org/) `scanimage` command to communicate with the scanner hardware directly.

## Why This Exists

The CanoScan LiDE 110 (USB ID `04A9:1909`, GL124 chip) has been supported by SANE's `genesys` backend for years, but using it required the command line. This app provides a proper GUI so you can just plug in your scanner and scan.

## Prerequisites

### 1. Install Homebrew

If you don't already have Homebrew:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 2. Install SANE Backends

```bash
brew install sane-backends
```

### 3. Connect Your Scanner

Plug the CanoScan LiDE 110 into a USB port. If you're using a USB-C Mac, you'll need a USB-A to USB-C adapter or hub.

### 4. Verify Scanner Detection (Optional)

You can confirm SANE sees your scanner from the terminal:

```bash
scanimage -L
```

You should see output like:

```
device `genesys:libusb:001:004' is a Canon LiDE 110 flatbed scanner
```

## Installing the App

### Option A: Build in Xcode (Recommended)

1. Open `CanoScanLiDE110.xcodeproj` in Xcode
2. Select **Product > Archive** from the menu bar
3. In the Organizer window, click **Distribute App**
4. Choose **Custom > Copy App**
5. Save the `.app` file, then drag it into your `/Applications` folder

### Option B: Build from the Command Line

```bash
xcodebuild -project CanoScanLiDE110.xcodeproj \
  -scheme CanoScanLiDE110 \
  -configuration Release \
  -derivedDataPath build

# Copy to Applications
cp -R "build/Build/Products/Release/CanoScan LiDE 110.app" /Applications/
```

### Option C: Run Directly from Xcode

1. Open `CanoScanLiDE110.xcodeproj` in Xcode
2. Click the Run button (or press Cmd+R)
3. The app launches directly — no install needed

### First Launch

Since the app isn't signed with an Apple Developer ID, macOS Gatekeeper will block it on first launch. To open it:

1. Right-click (or Control-click) the app in Finder
2. Select **Open** from the context menu
3. Click **Open** in the dialog that appears

You only need to do this once.

## Usage

### Scanning

1. Launch the app. If SANE is installed and the scanner is connected, you'll see the main interface. Otherwise, the app shows a setup guide.
2. Configure your scan settings in the toolbar:
   - **DPI**: 75, 150, 300 (default), 600, 1200, or 2400
   - **Mode**: Color, Grayscale, or Black & White
   - **Format**: PNG (default), JPEG, TIFF, or PDF
3. Click the **Scan** button in the toolbar, or press **Cmd+Return**
4. Wait for the scan to complete. A progress overlay appears with a Cancel button.

### Multi-Page Scanning

Each scan appends a new page to the sidebar on the left. This lets you scan multiple pages for a single document:

1. Place the first page on the scanner and press Cmd+Return
2. Replace with the next page and press Cmd+Return again
3. Repeat for all pages
4. Export as a single multi-page PDF (see below)

Click any page thumbnail in the sidebar to preview it. Right-click a page to delete it.

### Exporting

- **Export Current Page**: File > Export Current Page (Cmd+E) — saves the selected page in your chosen format (PNG, JPEG, TIFF, or PDF)
- **Export All as PDF**: File > Export All as PDF (Cmd+Shift+E) — merges all scanned pages into a single PDF document

You can also access both export options from the export button (arrow icon) in the toolbar.

### Keyboard Shortcuts

| Action | Shortcut |
|---|---|
| Scan a page | Cmd+Return |
| Export current page | Cmd+E |
| Export all pages as PDF | Cmd+Shift+E |

## How It Works

The app does not communicate with the scanner directly. Instead, it runs `scanimage` (installed by `brew install sane-backends`) as a subprocess:

1. **Detection**: Runs `scanimage -L` to check if the scanner is connected
2. **Scanning**: Runs `scanimage --format=tiff --resolution=300 --mode=Color` (etc.) and captures the raw TIFF data from stdout
3. **Conversion**: Converts the TIFF to your chosen output format using AppKit and PDFKit — no extra dependencies needed

This subprocess approach is intentional. If the USB communication has issues (which can happen with scanner drivers), only the `scanimage` process is affected — the app itself stays responsive and you can cancel and retry.

## Troubleshooting

### "SANE backends not found"

Make sure `scanimage` is installed:

```bash
brew install sane-backends
which scanimage
```

The app checks both `/opt/homebrew/bin/scanimage` (Apple Silicon) and `/usr/local/bin/scanimage` (Intel).

### "Scanner Not Detected"

- Check that the scanner is plugged in via USB and powered (the LiDE 110 is bus-powered, so it just needs a USB connection)
- Try a different USB port or adapter
- Run `scanimage -L` in Terminal to see if SANE detects it
- Unplug and replug the USB cable, then click Refresh in the app

### Scan hangs or fails

- Try a lower resolution (300 DPI instead of 2400)
- Cancel the scan, disconnect and reconnect the scanner, then try again
- Check Terminal for errors: `scanimage --format=tiff --resolution=300 --mode=Color > test.tiff`

### App won't open ("damaged" or "unidentified developer")

Right-click the app and choose **Open**, then click **Open** in the dialog. Alternatively:

```bash
xattr -cr "/Applications/CanoScan LiDE 110.app"
```
